# РГР за 2ой семестр по дисциплине "Программирование"

## Описание проекта
Проект — расчетно‑графическая работа на C++ по теме криптографических алгоритмов для шифрования и дешифрования файлов.
Реализованы три шифра:
- Атбаш (отражение байтов 0..255)
- Аффинный (C = a·P + b mod 256)
- XOR (P_i XOR K_{i mod n})

Приложение консольное, работает с любыми бинарными файлами, умеет получать/генерировать ключи и сохранять результаты. Проект предназначен для учебной демонстрации принципов криптографии.

## Цели
1. Реализовать классические криптоалгоритмы для бинарных данных.
2. Сделать удобный консольный интерфейс с обработкой ошибок.
3. Показать эволюцию методов шифрования от исторических до современных.

## Возможности
- Выбор режима (шифрование/дешифрование).
- Выбор шифра (Атбаш, Аффинный, XOR).
- Загрузка файла для обработки.
- Ввод, чтение из файла или генерация ключа.
- Индикатор прогресса обработки.
- Сохранение результата и ключа в файл.

## Требования
- Linux или macOS
- `g++` с поддержкой C++17

## Сборка
```bash
g++ -std=c++17 -O2 -Wall -Wextra -pedantic main.cpp ciphers.cpp file_io.cpp get_keys.cpp utils.cpp -o encodecs
```

## Запуск
```bash
./encodecs
```

## Быстрый старт
1. Запустите программу: `./encodecs`.
2. Выберите режим `1` (Шифрование).
3. Выберите шифр:
   - `1` Атбаш
   - `2` Аффинный
   - `3` XOR
4. Укажите путь к файлу (пример: `Files/text.txt`).
5. Выберите источник ключа:
   - Атбаш: ключ не требуется.
   - Аффинный: введите `a` (нечётное) и `b` (0..255) или сгенерируйте.
   - XOR: задайте длину ключа и байты ключа, либо сгенерируйте.
6. Подтвердите операцию и сохраните результат в файл.

## Подсказки по ключам
- Атбаш: ключ не используется.
- Аффинный: `a` должно быть нечётным (взаимно простым с 256).
- XOR: ключ повторяется по циклу, поэтому лучше использовать длину ключа > 1.
